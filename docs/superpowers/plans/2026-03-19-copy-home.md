# Copy Home Files Implementation Plan

> **For agentic workers:** REQUIRED: Use
> superpowers:subagent-driven-development (if subagents available)
> or superpowers:executing-plans to implement this plan. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace static `common/homedir/` with a mechanism that
copies user-specified files from the host home directory into
container and VM environments.

**Architecture:** A shared POSIX sh library reads a spec file
listing paths/globs to copy, translates them into rsync filter
rules, and rsyncs matched files to a destination. The container
path builds a tarball on the host and extracts it at startup. The
VM path rsyncs directly over SSH on first connect and on demand
via `agent-vm refresh-home`.

**Tech Stack:** POSIX sh, rsync, tar, bash (existing scripts)

**Spec:**
`docs/superpowers/specs/2026-03-19-copy-home-design.md`

### Task 2: Create the shared library

**Files:**

- Create: `common/scripts/copy-home-lib.sh`

- [ ] **Step 1: Create `copy-home-lib.sh`**

Write a POSIX sh library (`#!/bin/sh`) that exposes two functions:

`build_rsync_filters` - reads the spec file and a source home
directory, outputs rsync filter arguments (one per line) to
stdout. For each spec entry:

1. Skip comments (`#`) and blank lines
2. For exclusion lines (`!`), strip the `!` prefix and generate
   `- <pattern>` rules (with `- <pattern>/` and
   `- <pattern>/**` if the path is a directory in the source)
3. For inclusion lines, expand globs relative to the source
   home. For each resolved path:
   - If it is a directory: generate `+ <path>/` and
     `+ <path>/**`, plus `+ <parent>/` for every intermediate
     parent directory
   - If it is a file: generate `+ <path>`, plus `+ <parent>/`
     for every intermediate parent directory
4. After all entries, output `- *`
5. Deduplicate filter lines (intermediate parents may repeat)

`copy_home_files` - takes three arguments: spec file path,
source home directory, destination directory. Calls
`build_rsync_filters` to get the filter list, then runs:

```sh
rsync -a --relative --filter="merge <filter-file>" \
    "$src_home/./" "$dest_dir/"
```

The function writes the filters to a temp file, runs rsync, and
cleans up the temp file.

Both functions must be POSIX sh. No bashisms. `local` is allowed.

Add a function that builds the homedir tarball:

```bash
function build_homedir_tarball {
    local spec_file="$SCRIPT_DIR/../common/homedir-files-to-copy.txt"
    local staging_dir
    staging_dir=$(mktemp -d)

    if [[ ! -f "$spec_file" ]]; then
        echo "Warning: $spec_file not found, skipping home copy"
        echo "$staging_dir"
        return 0
    fi

    # Use the shared library to copy matched files to staging
    copy_home_files "$spec_file" "$HOME" "$staging_dir"

    # Create tarball from staging
    tar -czf "$staging_dir/homedir.tar.gz" \
        -C "$staging_dir" \
        --exclude=homedir.tar.gz .

    # Remove the staged files, keep only the tarball
    find "$staging_dir" -mindepth 1 -maxdepth 1 \
        ! -name homedir.tar.gz -exec rm -rf {} +

    echo "$staging_dir"
}
```

Add a function that rsyncs spec'd files to the VM:

```bash
function sync_home_to_vm {
    local spec_file="$SCRIPT_DIR/../common/homedir-files-to-copy.txt"

    if [[ ! -f "$spec_file" ]]; then
        echo "Warning: $spec_file not found, skipping home copy"
        return 0
    fi

    echo "Syncing home directory files to VM..."

    # Build rsync filter file
    local filter_file
    filter_file=$(mktemp)
    build_rsync_filters "$spec_file" "$HOME" > "$filter_file"

    # Rsync to VM over SSH
    rsync -a --relative \
        --filter="merge $filter_file" \
        -e "ssh -F $SSH_CONFIG -o StrictHostKeyChecking=no" \
        "$HOME/./" "${VM_HOST}:$(get_vm_home)/"

    rm -f "$filter_file"
    echo "✓ Home directory files synced to VM"
}
```
